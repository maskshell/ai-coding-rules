# 安全规范

## 输入验证

### 外部输入

- 所有用户输入必须验证类型、长度、格式、范围
- 使用参数化查询防止 SQL 注入
- 对 HTML 输出进行转义防止 XSS

### API 安全

- 验证请求来源和身份
- 限制请求频率
- 敏感操作需要双重验证

## 敏感信息

### 密钥管理

- 密钥、密码、令牌必须存在环境变量或密钥管理服务中
- 绝不提交密钥到版本控制
- 使用 .gitignore 防止敏感文件被提交

### 数据保护

- 敏感数据（密码、个人信息）必须加密存储
- 日志中不输出敏感信息
- 传输层使用 HTTPS/TLS

## 代码安全

### 依赖安全

- 定期检查依赖漏洞（npm audit, pip check）
- 及时更新有漏洞的依赖

### 访问控制

- 实现最小权限原则
- 敏感操作需要权限检查
