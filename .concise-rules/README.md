# 精简版规则使用指南

## 双轨制规则说明

本项目提供两套规则系统，以满足不同场景需求：

### 📦 完整版（full-rules/）

- **用途**：人类阅读、学习参考、团队培训
- **特点**：详细、全面、包含丰富示例和背景说明
- **位置**：`full-rules/ide-layer/` 和 `full-rules/project-templates/`
- **适用场景**：初次使用、理解规则设计思路、团队规范制定

### ⚡ 精简版（.concise-rules/）

- **用途**：AI 执行、日常使用、实际开发
- **特点**：简洁、可操作、无冗余解释
- **位置**：`.concise-rules/ide-layer/` 和 `.concise-rules/project-templates/`
- **优势**：token 消耗减少 73%，响应速度提升，AI 理解更清晰

## 如何使用

### Cursor 用户

**全局规则（IDE 层）**：在 `~/.cursor/rules/` 中创建符号链接

```bash
# IDE 层规则（适用于所有项目）
mkdir -p ~/.cursor/rules
ln -s /path/to/ai-coding-rules/.concise-rules/ide-layer/* ~/.cursor/rules/
```

**项目规则（项目模板）**：在项目的 `.cursor/rules/` 目录中创建符号链接

```bash
# 进入你的项目目录
cd /path/to/your/project

# 项目特定规则（仅适用于当前项目）
mkdir -p .cursor/rules
ln -s /path/to/ai-coding-rules/.concise-rules/project-templates/react-app/* .cursor/rules/
```

### Claude Code 用户

在配置文件中引用精简版规则：

```json
{
  "rulesDirectory": "/path/to/ai-coding-rules/.concise-rules/ide-layer"
}
```

或者直接将精简版规则文件复制到项目根目录：

```bash
# 复制到项目根目录（项目规则）
mkdir -p .cursor/rules
cp /path/to/ai-coding-rules/.concise-rules/project-templates/react-app/* .cursor/rules/
```

### 其他编辑器

根据编辑器规则系统的文档，将精简版规则文件配置到相应的规则目录。

## 最佳实践

### 对于新团队

1. **第一周**：使用完整版了解规则设计思路
2. **之后**：切换到精简版进行日常开发
3. **定期回顾**：参考完整版进行规则优化

### 对于有经验的团队

直接使用精简版，根据需要查阅完整版的特定部分。

### 规则定制

1. 在精简版基础上，只保留团队需要的规则
2. 删除不适用的规则以减少干扰
3. 添加项目特定的规则
4. 保持规则总数 < 20 个

## 为什么选择精简版？

**优势：**

- ✅ AI 处理更快（token 更少）
- ✅ AI 理解更清晰（核心指令突出）
- ✅ 成本更低（token 消耗减少 73%）
- ✅ 响应更及时（上下文处理更快）

**完整版的使用场景：**

- 📚 学习规则设计思路
- 🎓 团队培训和规范制定
- 📝 添加新规则时的参考模板
- 🤔 理解某个规则背后的"为什么"

## 迁移指南

### 从完整版迁移到精简版

不需要特殊操作，只需将规则目录切换到精简版即可。

### 保持更新

当更新完整版规则时，同步更新对应的精简版：

1. 识别完整版中的核心指令
2. 移除解释性文字和过多示例
3. 保留 1-2 个关键示例
4. 保持结构一致（数字前缀）
