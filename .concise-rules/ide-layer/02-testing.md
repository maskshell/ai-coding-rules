# 测试规范（精简版）

## 测试覆盖率

**必须测试的场景：**

- 核心业务逻辑
- 边界条件和边界值
- 错误处理路径
- 关键用户流程

**覆盖率目标：**

- 核心业务模块：≥ 80%
- 工具函数：≥ 90%
- UI组件：≥ 70%

## 测试类型

**单元测试：**

- 每个函数/方法都应该有对应测试
- 测试用例名描述测试场景（AAA 模式）

**集成测试：**

- 测试模块之间的交互
- 测试外部依赖（数据库、API）

## 测试实践

- 测试代码和生产代码同等重要
- 避免测试之间的依赖
- 保持测试的独立性
- 失败测试应立即修复
- 删除过期和无用的测试
